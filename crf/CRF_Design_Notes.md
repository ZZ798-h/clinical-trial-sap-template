# Case Report Form Design Notes

## Purpose

Case Report Forms (CRFs) are used to collect study data in a structured and consistent way. A well-designed CRF supports data quality, reduces missing or ambiguous data, and helps ensure that the variables required for the Statistical Analysis Plan are captured correctly.

## Relationship Between CRF and SAP

The CRF and SAP are closely connected but serve different purposes:

- The CRF defines what data are collected.
- The SAP defines how collected data will be analyzed.
- The EDC/database structure stores CRF data for cleaning, query management, and statistical analysis.
- Well-designed CRFs can reduce data queries and improve downstream analysis efficiency.

## Key CRF Design Considerations

- Align CRF fields with protocol objectives and endpoints
- Capture variables required for primary and secondary analyses
- Use clear variable definitions and response options
- Minimize free-text fields when structured responses are possible
- Include data validation rules when appropriate
- Avoid collecting unnecessary data
- Support traceability between protocol, CRF, database, and SAP
